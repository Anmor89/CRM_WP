# Публикация West Part CRM в интернет (GitHub + GitHub Pages)

Эта инструкция нужна **один раз** — чтобы настроить публикацию.
Дальше каждое обновление выкладывается одним двойным кликом.

Что получится в итоге: сайт CRM будет доступен по постоянной ссылке вида
`https://ВАШ-ЛОГИН.github.io/west-part-crm/`, и при каждой публикации там
будет автоматически разворачиваться актуальная версия.

---

## Шаг 0. Удалить служебную папку `.git`

В папке `West Part CRM` могла появиться скрытая папка `.git` (я создал её
при настройке, но из-за прав доступа не смог дочистить).

1. Откройте папку `West Part CRM` в проводнике.
2. Включите показ скрытых файлов: вкладка «Вид» → галочка «Скрытые элементы».
3. Если видите папку `.git` — **удалите её** (Shift+Delete).

Если папки `.git` нет — пропустите этот шаг.

---

## Шаг 1. Установить Git (если ещё не установлен)

1. Откройте PowerShell и введите:
   ```
   git --version
   ```
2. Если показалась версия (например `git version 2.43`) — Git уже есть,
   переходите к Шагу 2.
3. Если ошибка — установите Git:
   - Вариант А: в PowerShell выполнить `winget install Git.Git`
   - Вариант Б: скачать с https://git-scm.com/download/win и установить
     (все настройки по умолчанию)
4. После установки **закройте и заново откройте PowerShell**.

---

## Шаг 2. Создать репозиторий на GitHub

1. Зайдите на https://github.com (при необходимости — зарегистрируйтесь).
2. Нажмите зелёную кнопку **New** (создать репозиторий).
3. Заполните:
   - **Repository name:** `west-part-crm`
   - **Public** (обязательно — для бесплатного GitHub Pages нужен публичный)
   - Галочки «Add a README», «.gitignore», «license» — **НЕ ставить**
4. Нажмите **Create repository**.
5. На открывшейся странице скопируйте адрес репозитория, он вида:
   `https://github.com/ВАШ-ЛОГИН/west-part-crm.git`

---

## Шаг 3. Первая публикация (выполняется один раз)

Откройте PowerShell и выполните команды по очереди.

**Важно:**
- путь в первой команде `cd` — это путь к вашей папке `West Part CRM`
  (например `C:\Users\anton\YandexDisk\Антон\Claude Cowork\West Part CRM`);
- в команде `remote add` подставьте свой адрес репозитория из Шага 2;
- две команды `git config` (имя и почта) обязательны — без них git
  не даст сделать коммит.

```powershell
cd "C:\Users\anton\YandexDisk\Антон\Claude Cowork\West Part CRM"

git init
git branch -M main
git config user.name "Anton Morochko"
git config user.email "anton.morochko@gmail.com"
git add .
git commit -m "West Part CRM — первая версия"
git remote add origin https://github.com/ВАШ-ЛОГИН/west-part-crm.git
git push -u origin main
```

При первом `git push` откроется окно входа в GitHub — войдите через браузер
(кнопка «Sign in with your browser»). Это нужно сделать один раз, дальше
вход запомнится.

После успешного `push` обновите страницу репозитория на GitHub — там должны
появиться файлы `index.html`, `README.md` и т.д.

> Если папка переедет (например, в Яндекс.Диск) — ничего настраивать заново
> не нужно, git привязан к папке, а не к её расположению.

---

## Шаг 4. Включить GitHub Pages (сайт в интернете)

1. На странице репозитория откройте **Settings** (шестерёнка сверху).
2. В левом меню выберите **Pages**.
3. В разделе **Build and deployment → Source** выберите **Deploy from a branch**.
4. Ниже: ветка **main**, папка **/ (root)**. Нажмите **Save**.
5. Подождите 1–2 минуты и обновите страницу — вверху появится ссылка вида:
   `https://ВАШ-ЛОГИН.github.io/west-part-crm/`

Это и есть адрес вашей CRM в интернете. Откройте — приложение запустится.

> Прямой адрес можно дать так: `https://ВАШ-ЛОГИН.github.io/west-part-crm/index.html`

---

## Шаг 5. Как публиковать обновления (каждый раз)

После того как я (Claude) обновлю `index.html` с новыми доработками:

1. Зайдите в папку `West Part CRM`.
2. Правый клик по файлу **`deploy.ps1`** → **«Выполнить с помощью PowerShell»**.
3. Скрипт спросит описание изменений — можно просто нажать Enter.
4. Готово. Через 1–2 минуты сайт по вашей ссылке обновится автоматически.

Всё. Никаких команд вручную больше вводить не нужно.

> Если Windows не даёт запустить скрипт («выполнение скриптов отключено») —
> один раз выполните в PowerShell:
> `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`
> и подтвердите «Y».

## Частые вопросы

**Где хранятся версии?**
Каждая публикация — это отдельный коммит в git. Полная история всех версий
видна на GitHub во вкладке нужного файла → кнопка «History». Папка `backups`
для git не нужна и в репозиторий не отправляется (она в `.gitignore`).

**Данные клиентов попадут на GitHub?**
Нет. В репозитории только код приложения. Данные (клиенты, события) каждый
пользователь хранит локально в своём браузере. Это особенность прототипа —
когда перейдём на полноценное веб-приложение, данные будут на сервере в общей
базе.

**Сайт публичный — любой может зайти?**
Да, по ссылке зайти может любой, но без логина внутрь не попадёт. Для прототипа
этого достаточно. На боевом этапе закроем доступ полностью.

**Можно ли своё доменное имя?**
Да, GitHub Pages поддерживает свой домен — настроим, когда понадобится.
